Rabin-Karp Algorithm

  • Algorithm
    • Text = 1234, pattern = 34, then t0 = 12, t1 = 23, t2 = 34.  The idea is to calculate tn from tn-1.  For example, t1 =  t0%10 * 10 + 3
    • For lengthy patterns,  tn can be too large.  However, we can compare its remainder of mod operation, but it may have a spurious hit (false positive), we should compare the entire string whenever we have a spurious hit   

 

  • Expected running time = O(n).  Worst case running time = O(n * m), if we had spurious hit in every digit
    • n = length of full text
    • m = legnth of pattern
