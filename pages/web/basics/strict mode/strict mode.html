Strict Mode

  • Strict mode was Introduced in EMCAScript 5, makes some changes to JavaScript semantics
  • Changes
    • Eliminates some silent errors by throwing errors
      • 'use strict'
        v = "assign to a undeclared variable" // throw exception
    • Fixes mistakes that make it difficult for JavaScript engines to perform optimizations
      • strict mode disallow "with" statement which is ambiguous
    • Prohibits some syntax likely to be defined in future versions of ECMAScript
      • 'use strict';
        var implements; // throw exception
