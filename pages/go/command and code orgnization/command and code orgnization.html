Command on Single File

  • build command

    • Usage: go build main.go

    • Output: main.exe

  • run command

    • Usage: go run main.go

    • Effect: build and run

    • Output: main.exe and execute main.exe

  • fmt command

    • Usage: go fmt main.go

    • Effect: Formtting your code to have a consistent layout, for example, spacing

    • Output: main.go

  • vet command

    • Usage: go vet main.go

    • Effect: Statically analysis your code, i.e. lint

    • Output: Error message (if any)

  • help command

    • Usage: go help [command]

    • Example: go help build

  • List of all commands

    • bug start a bug report
      build compile packages and dependencies
      clean remove object files and cached files
      doc show documentation for package or symbol
      enc print Go environment information
      fix update packages to use new APIs
      fmt gofmt (reformat) package sources
      genereate generate Go files by processing source
      get add dependencies to current module and install them
      install compile and install packages and dependencies
      list list packages or modules
      mod module maintenance
      run compile and run Go program
      test test packages
      tool run specified go tool
      version print Go version
      vet report likely mistakes in packages

    • Reference: https://golang.org/cmd/go/

Command on Package

  • Steps if you want to apply command on packpages

    1. Usage: go [cmd] [packages]

    2. Follow the go code orgnization rule like below.  Put all your source files in Dir\src

      • Dir

        • bin

        • pkg

        • src

          • package1

            • hello1.go

          • package 2

            • hello2.go

          • main.go

    3. Add "Dir" (say D:\my_go_dir) to gopath.  for example, set gopath=%gopath%;D:\my_go_dir

    4. Example: go build main package1 package2
